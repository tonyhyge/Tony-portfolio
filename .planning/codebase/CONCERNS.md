# Codebase Concerns

**Analysis Date:** 2026-06-19

## Critical: No Actual Portfolio Content

**Entire application is still the default Next.js template:**
- `src/app/page.tsx` — Default "To get started, edit the page.tsx file" page (66 lines)
- `src/app/layout.tsx` — Metadata still says "Create Next App" and "Generated by create next app"
- `public/` — Contains only default Next.js placeholder SVGs (next.svg, vercel.svg, globe.svg, etc.)
- No portfolio sections, no hero, no about, no experience, no projects

**Files:** `src/app/page.tsx`, `src/app/layout.tsx`, `public/`
**Impact:** The site deployed to GitHub Pages is a generic Next.js template with zero personal content
**Fix approach:** Execute ROADMAP.md phases 1.1 through 1.7 to build the MVP portfolio

## Missing Configuration Files

**No `.gitignore`:**
- Risk of committing `node_modules/`, `.next/`, `out/`, `.env` files
- Repository already contains some files that should be ignored

**No `tsconfig.json` visible:**
- TypeScript strict mode may not be properly configured
- Path aliases (`@/`) may not be set up

**No ESLint configuration file:**
- `package.json` has `"lint": "eslint"` script but no `.eslintrc.*` or `eslint.config.*`
- Linting will fail or use empty defaults

**Files:** root directory
**Impact:** Poor developer experience, risk of committing build artifacts
**Fix approach:** Add `.gitignore`, `tsconfig.json` with path aliases, ESLint config

## Zero Test Coverage

**No test files exist:**
- No `*.test.*` or `*.spec.*` files in `src/`
- No test framework configured (no `jest.config.*`, `vitest.config.*`, `playwright.config.*`)
- No E2E tests despite Playwright being recommended

**Files:** entire `src/` tree
**Risk:** No regression safety when building portfolio features
**Priority:** High

## Unused Dependencies and Bloat

**`shadcn` package installed but barely used:**
- Only 7 shadcn components installed in `src/components/ui/` (button, card, sheet, navigation-menu, avatar, badge, separator)
- None of these are actually imported or used in `page.tsx`

**`threejs-skills/` directory:**
- Contains a full git submodule or cloned repo with Three.js skill references
- Not imported or referenced anywhere in the application
- Adds unnecessary repository size

**`ui-ux-pro-max-skill/` directory:**
- Large external skill reference directory (~5 top-level dirs)
- Contains screenshots, CLI tools, docs — none used by the application
- Bloated repository with irrelevant content

**Files:** `package.json`, `threejs-skills/`, `ui-ux-pro-max-skill/`
**Impact:** Larger bundle, longer install times, confusing repo structure
**Fix approach:** Remove unused directories, audit and prune unused dependencies

## Fragile: Hardcoded GitHub Path

**`next.config.ts` hardcodes repository name:**
```typescript
basePath: process.env.NODE_ENV === "production" ? "/Tony-portfolio" : "",
```
- If repository is renamed or forked, build will break
- No `.env` file or configuration abstraction for this value

**File:** `next.config.ts`
**Fix approach:** Use an environment variable `NEXT_PUBLIC_BASE_PATH` or make configurable

## Missing Content Infrastructure

**No content directory or MDX support:**
- CLAUDE.md references `/content/blog/` and `/content/research/` directories
- Neither directory exists
- No MDX packages installed (`@next/mdx`, `next-mdx-remote`, etc.)
- Blog and research features (Milestone 2) will require significant setup

**Impact:** Cannot add blog posts or research projects as specified in ROADMAP.md
**Priority:** Medium (blocks Milestone 2)

## Performance: Image Optimization Disabled

**`next.config.ts` disables image optimization:**
```typescript
images: {
  unoptimized: true,
}
```
- Required for static export but means all images load at full resolution
- No image optimization, resizing, or format conversion
- Profile photo and project images will load uncompressed

**File:** `next.config.ts`
**Impact:** Slower page loads, especially on mobile
**Improvement path:** Consider using a CDN for static assets or pre-optimized images

## No Error Boundaries

**No error handling infrastructure:**
- No `error.tsx` or `global-error.tsx` files in `src/app/`
- No error boundaries for React components
- Runtime errors will crash the entire page with no fallback

**Files:** `src/app/`
**Fix approach:** Add Next.js error boundary files per route segment

## Accessibility: No ARIA or Semantic Structure

**Default template lacks accessibility:**
- `page.tsx` uses generic `<div>` wrappers instead of semantic HTML
- No `aria-label`, `aria-labelledby`, or role attributes
- No skip-to-content link
- No keyboard navigation support

**File:** `src/app/page.tsx`
**Fix approach:** Use semantic elements (`<header>`, `<main>`, `<nav>`, `<section>`) per web/coding-style.md

## Security: External Links Lack Complete Protection

**External links have `rel="noopener noreferrer"` but no CSP:**
- No Content Security Policy configured
- No security headers (HSTS, X-Frame-Options, etc.)
- No referrer policy set

**Files:** `src/app/page.tsx`, `next.config.ts`
**Recommendations:** Add security headers via `next.config.ts` headers config

## Dependency Risks

**Next.js 16.2.9 with React 19.2.4:**
- Using very recent versions — potential for undiscovered bugs
- No version lock pinning (uses `^` ranges)
- `shadcn@^4.11.0` is a major version that may have breaking changes

**File:** `package.json`
**Risk:** Build may break on minor version bumps

## Missing Critical Features

**No 404 page:**
- No `src/app/not-found.tsx`
- Users hitting invalid routes see a generic Next.js error

**No robots.txt or sitemap:**
- Search engine optimization not configured
- Required for public-facing portfolio

**No favicon customization:**
- Still using default Next.js favicon.ico

**Files:** `src/app/`
**Priority:** Medium (should be done before public launch)

## Repository Structure Concerns

**`public/` contains only default SVGs:**
- No portfolio images, icons, or assets
- All files are generic Next.js placeholders that should be replaced or removed

**Shadcn components installed but unused:**
- `button.tsx`, `card.tsx`, `sheet.tsx`, etc. are all dead code until the portfolio is built
