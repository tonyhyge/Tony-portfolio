# Phase 1.7 — UI Design Contract

> Visual and interaction contract for the polish-and-deploy phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

This phase introduces minimal new UI: a single custom 404 page (`not-found.tsx`). All other work is quality-assurance infrastructure (tests, audits, CI) that produces no user-visible UI. The 404 page reuses the existing design system without adding new components or dependencies.

## Interaction States

The 404 page has minimal interactivity -- only the "Back to Home" link/button.

| Element | States | Implementation |
|---------|--------|----------------|
| Back to Home button | Default, hover, focus-visible, active | Use existing shadcn `Button` with `variant="default"` (accent bg). Hover: `bg-accent/90` via shadcn. Focus: `outline-ring/50` via globals.css base layer. |
| Hexagon decoration | Static (no animation) | SVG inline with gradient stroke. Purely decorative (`aria-hidden`). |

No destructive actions, form inputs, loading states, or toggle interactions in this phase.

---

## Accessibility Notes (404 Page)

- Hexagon is `aria-hidden` (decorative only) -- consistent with HexagonBadge pattern in Hero
- Heading is an `<h1>` for correct document outline
- Body text uses `text-muted-foreground` (meets 4.5:1 contrast ratio per existing token audit)
- CTA is a semantic `<a>` or `<button>` (keyboard accessible by default)
- Page responds to `.dark` class toggle (inherits from root layout)
