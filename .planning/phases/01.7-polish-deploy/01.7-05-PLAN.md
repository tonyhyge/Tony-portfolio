<task type="auto">
  <name>Task 2: Add audit job after deploy with Lighthouse + a11y gates</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Continue editing `.github/workflows/deploy.yml` to add an `audit` job that runs after `deploy` per D-09. Add after the `deploy` job:

    ```yaml
    audit:
      runs-on: ubuntu-latest
      needs: deploy
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: npm
        - run: npm ci
        - run: npm run build

        - name: Run Lighthouse CI
          run: npx lhci autorun
          env:
            LHCI_GITHUB_APP_TOKEN: ${{ secrets.LHCI_GITHUB_APP_TOKEN }}

        - name: Run a11y checks
          run: |
            npx serve@latest ./out -l 3000 --no-clipboard &
            sleep 3
            npx playwright install chromium --with-deps
            npx playwright test tests/e2e/a11y.spec.ts --reporter=github
    ```

    Key details:
    - Audit runs after deploy, so `needs: deploy`
    - Re-builds the static export for Lighthouse CI (needs ./out/)
    - `lhci autorun` reads lighthouserc.json, asserts all scores >= 0.9 per D-10. If any score < 0.9, the step fails (blocking per D-08).
    - a11y tests run via Playwright against the same static export served locally, using `@axe-core/playwright` with WCAG AA tags. If violations found, step fails.
    - The `serve` command uses `--no-clipboard` per Pitfall 2.
    - Playwright installs chromium with system deps (`--with-deps`) for the Ubuntu CI runner.
  </action>
  <verify>
    <automated>grep -q "audit:" .github/workflows/deploy.yml && grep -q "lhci" .github/workflows/deploy.yml && grep -q "a11y.spec" .github/workflows/deploy.yml && echo "pass" || echo "fail"</automated>
  </verify>
  <done>
    audit job added after deploy. Runs Lighthouse CI with score >= 0.9 assertions and axe-core WCAG AA audits. Deployment blocked if either fails.
  </done>
</task>
