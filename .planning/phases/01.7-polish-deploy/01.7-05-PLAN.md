<task type="auto">
  <name>Task 2: Add audit job between build and deploy with Lighthouse + a11y gates</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
    Continue editing `.github/workflows/deploy.yml` to add an `audit` job that runs between `build` and `deploy`. This way audit gates deploy per D-08 (scores < 90 or a11y violations block deploy).

    Add this job after the `build` job and before `deploy`:

    ```yaml
    audit:
      runs-on: ubuntu-latest
      needs: build
      steps:
        - uses: actions/download-artifact@v4
          with:
            name: build-output
            path: ./out

        - uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: npm

        - run: npm ci

        - name: Run Lighthouse CI
          run: npx lhci autorun --staticDistDir=./out
          env:
            LHCI_GITHUB_APP_TOKEN: ${{ secrets.LHCI_GITHUB_APP_TOKEN }}

        - name: Run a11y checks
          run: |
            npx serve@latest ./out -l 3000 --no-clipboard &
            sleep 3
            npx playwright install chromium --with-deps
            npx playwright test tests/e2e/a11y.spec.ts --reporter=github
    ```

    Update the `deploy` job to add `needs: audit`.

    Key details:
    - Audit runs against the downloaded build artifact (reuses `build` output, no rebuild needed)
    - `lhci autorun --staticDistDir=./out` runs against the static export directory
    - If any Lighthouse score < 0.9 or a11y violations found, the step fails and `deploy` is blocked per D-08
    - The `serve` command uses `--no-clipboard` per Pitfall 2
    - Playwright installs chromium with system deps (`--with-deps`) for the Ubuntu CI runner
    - `download-artifact@v4` must match the upload artifact action version in the `build` job
  </action>
  <verify>
    <automated>grep -q "audit:" .github/workflows/deploy.yml && grep -q "lhci" .github/workflows/deploy.yml && grep -q "a11y.spec" .github/workflows/deploy.yml && grep -q "needs: audit" .github/workflows/deploy.yml && echo "pass" || echo "fail"</automated>
  </verify>
  <done>
    audit job added between build and deploy. Runs Lighthouse CI with score >= 0.9 assertions and axe-core WCAG AA audits against the built artifact. deploy job has `needs: audit`. Deployment blocked if audit fails.
  </done>
</task>
