<decisions>
## Implementation Decisions

### Testing Infrastructure
- **D-01:** **Vitest** for unit + component tests (not Jest) — native ESM, Next.js compatible, community standard
- **D-02:** **Playwright** for E2E tests — screenshot key pages at 320, 768, 1024, 1440 breakpoints; verify nav, section loading, dark mode toggle
- **D-03:** **Testing Library** (@testing-library/react, @testing-library/jest-dom, @testing-library/user-event) for component tests
- **D-04:** **Coverage target: 80%** — line + branch coverage enforced for utility code (`src/lib/`); component tests covering existence and key states; E2E covering critical paths
- **D-05:** Tests colocated with source files (`{module}.test.ts`, `{Component}.test.tsx` for units/components; `{page}.spec.ts` for E2E)
- **D-06:** Test data factories/fixtures in `src/__fixtures__/` for shared data, colocated for component-specific

### CI Pipeline
- **D-07:** CI runs tests on every push to main — **gate on failures + coverage** (build fails if tests fail or coverage < 80%)
- **D-08:** CI runs Lighthouse CI and a11y checks on push — gate deploy if scores < 90 in any category or WCAG AA violations found
- **D-09:** Existing deploy workflow updated: test job runs before build, audit job runs after deploy

### Performance Targets
- **D-10:** Lighthouse targets — **all 5 categories ≥ 90** (Performance, Accessibility, Best Practices, SEO, PWA-skip)
- **D-11:** Core Web Vitals: LCP < 2.5s, CLS < 0.1, TBT < 200ms, FCP < 1.5s
- **D-12:** Bundle budget: total JS < 150kb gzipped for landing page

### Accessibility
- **D-13:** **WCAG AA** compliance — automated checks (axe-core via Playwright or direct)
- **D-14:** Keyboard navigation — tab order, focus trap in Sheet, skip-to-content link
- **D-15:** Screen reader — ARIA labels, semantic HTML, alt text on images/icons
- **D-16:** Color contrast — minimum 4.5:1 ratio (use existing OKLCH tokens)
- **D-17:** NOT scoped for this phase: reduced-motion preferences, focus indicator visibility polish, prefers-reduced-motion animations

### 404 Page
- **D-18:** **Creative/playful** design — hexagon/brand-themed, not just a generic "not found"
- **D-19:** Clear "back to home" navigation
- **D-20:** Stays within existing design system (no new dependencies)

### SEO & Crawlers
- **D-21:** **robots.txt** — allow all AI crawlers (GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot) plus standard crawlers; maximize LLM discoverability
- **D-22:** **sitemap.xml** — dynamic sitemap covering `/` + all sections + `/research/<slug>` pages
- **D-23:** All content in English per Phase 1.1 decision

### Deployment
- **D-24:** Ship on **GitHub Pages URL** (`tonyhyge.github.io/Tony-portfolio`) — no custom domain for MVP
- **D-25:** Pre-flight: verify Formspree form submission works end-to-end after deploy
- **D-26:** First deploy triggered by pushing `main` and monitoring the GitHub Actions run

### Claude's Discretion
- Exact Lighthouse CI integration tool (`@lhci/cli` or GitHub Action `treosh/lighthouse-ci-action`)
- Accessibility audit tool (axe-core Playwright integration or standalone `@axe-core/cli`)
- 404 page exact visual design (hexagon motif, layout, animation if any)
- robots.txt exact policies per crawler agent
- sitemap.xml exact frequency/priority values
- Test file locations (colocated per D-05 convention, exact naming)
- Coverage reporting configuration (reporter format, thresholds per module)
- CI workflow optimization (dependency caching, parallel job execution)

</decisions>
