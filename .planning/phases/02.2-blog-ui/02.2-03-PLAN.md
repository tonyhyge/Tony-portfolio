  <name>Task 1: Series navigation banner</name>
  <files>
    src/components/blog/series-nav.tsx
  </files>
  <action>
    Create a SeriesNav component for series-aware posts. Rendered at both top and bottom of the article.

    **Props:**
    - post: BlogPostMeta (to read series name and determine position)
    - allPosts: BlogPostMeta[] (to find series siblings sorted by date)
    - position: "top" | "bottom" (allows different styling or spacing if needed)

    **Logic:**
    - If post.series is undefined, render nothing (return null)
    - Filter allPosts where p.series === post.series and p.slug !== post.slug (exclude self), then sort by date ascending
    - Also include the current post in the series list for counting: merge and sort by date
    - Determine current post index (0-based) in the sorted series list
    - N = index + 1 (1-based), M = seriesPosts.length

    **Visual:**
    - Border-left: 3px solid accent color — use border-l-3 border-accent
    - Background: bg-muted/50 (subtle)
    - Rounded corners: rounded-md
    - Padding: p-3
    - Layout: flex items-center justify-between

    **Content (left side):**
    - "Part N of M — Series Title" text
    - text-sm (Small token), text-muted-foreground
    - Series title in text-foreground font-medium

    **Navigation (right side):**
    - Previous: ChevronLeft icon (lucide-react), links to /blog/{prev.slug} — only if there's a previous post in series
    - Next: ChevronRight icon, links to /blog/{next.slug} — only if next exists
    - Icons rendered as Link components, with hover:text-accent and hover:scale-110 transition

    **Edge cases:**
    - Post is the only one in the series: show the banner but no nav arrows
    - Post is first in series: only next arrow visible
    - Post is last in series: only prev arrow visible
    - No series field: component returns null
  </action>

  <name>Task 3: Prev/Next post navigation</name>
  <files>
    src/components/blog/prev-next-nav.tsx
  </files>
  <action>
    Create a PrevNextNav component for adjacent post navigation at the bottom of the post page.

    **Props:**
    - post: BlogPostMeta (current post)
    - allPosts: BlogPostMeta[] (full sorted list for computing neighbors)

    **Logic (per D-07, per UI-SPEC):**
    - If post.series is defined:
      - Filter allPosts where p.series === post.series, sort by date ascending
      - Find current post index in this filtered list
      - Previous = element at index-1 (older in series)
      - Next = element at index+1 (newer in series)
    - If no series:
      - Use allPosts already sorted by date descending (from getAllPosts())
      - Find current post index
      - Previous = element at index+1 (chronologically older)
      - Next = element at index-1 (chronologically newer)
    - If a neighbor doesn't exist, that side is empty (render spacer)

    **Visual (per UI-SPEC):**
    - Layout: grid grid-cols-2 gap-6
    - Left column: "Previous" label (text-sm text-muted-foreground) + post title (text-base text-foreground)
      - Use ChevronLeft icon before the label
      - When no previous post: empty div (spacer)
    - Right column: "Next" label + post title, right-aligned
      - ChevronRight icon after the label
    - Each side wraps in a Link to /blog/{slug}
    - Hover: title color changes to accent, arrow translates 4px in its direction
    - group for hover behavior: group-hover:translate-x-1 (prev), group-hover:-translate-x-1 (next)

    **Edge cases:**
    - Single post: both sides empty, show nothing or just "No more posts"
    - First post (no newer): only next column rendered
    - Last post (no older): only prev column rendered
    - Series with one post: no prev/next, render null
  </action>
