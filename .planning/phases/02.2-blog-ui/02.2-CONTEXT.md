# Phase 2.2: Blog UI — Context

**Gathered:** 2026-06-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Blog public surface — list page at `/blog`, enhanced post page at `/blog/[slug]`, tag browsing, and post-to-post navigation. Includes: blog index with hero + grid layout, tag filtering (on-page + tag routes), post page polish (breadcrumb, cover image, TOC, series nav, share buttons), Prev/Next navigation, and table of contents for long posts.

Does NOT include: RSS feed generation (Phase 2.3), SEO per post (Phase 2.3), comment system (future), search/full-text search (future), bookmarking or read-later features (future).
</domain>

<decisions>
## Implementation Decisions

### Blog List Layout
- **D-01:** Cards + hero layout — the latest post displays as a large hero card with cover image, title, excerpt. Remaining posts render in a responsive 2-column grid.
- **D-02:** Hero card uses the post's `cover` image (from frontmatter). When no cover image exists, fall back to an accent gradient (cyan→violet) with title overlay — consistent with the hero section's gradient treatment.
- **D-03:** Grid cards: 2 columns on desktop, 1 column on mobile. Denser than hero but still readable with title, excerpt, date, and tags.

### Tag Filtering
- **D-04:** Dual approach — on-page filter via URL search params (`?tag=xyz`) AND separate route pages at `/blog/tag/[tag]` for SEO and link sharing.
- **D-05:** Tag pills displayed at top of the blog list page. Selecting a tag sets the URL param and filters posts client-side (or via server-side filter on re-render). Both approaches point to the same data, just different entry points.

### Previous / Next Navigation
- **D-06:** Displayed as full post titles at the bottom of the post page. "Previous" (newer) and "Next" (older) labels with the post title.
- **D-07:** Series-bound if the post has a `series` frontmatter field — only navigate within the same series. When no series is set, fall back to chronological adjacency.

### Table of Contents
- **D-08:** Adaptive — sticky sidebar on the right on desktop, collapsible TOC at the top of the article on mobile/tablet.
- **D-09:** Auto-generated by parsing h2/h3 heading tags from the MDX content. Support manual override via frontmatter `toc: [...]` field if the user wants to limit which headings appear.
- **D-10:** Active heading tracking — highlight the current section as the user scrolls, using IntersectionObserver (consistent with `useActiveSection` hook pattern).

### Post Page Polish
- **D-11:** Breadcrumb trail: `Blog > [tag or "Posts"] > [title]` at the top of the article.
- **D-12:** Back-to-list link ("← Back to all posts") below the breadcrumb.
- **D-13:** Cover image displayed at the top of the article (hero width) when available, with accent gradient fallback.
- **D-14:** Series navigation banner at both the top and bottom of the article — shows "Part N of M" with links to previous/next parts in the series.
- **D-15:** Share buttons: X/Twitter + LinkedIn + copy link. Placed at the bottom of the article, after the content but before Prev/Next navigation.

### Metadata Display
- **D-16:** Already implemented — date (via `formatPostDate`), reading time estimate. No changes needed to these.
- **D-17:** Tags displayed as pills on both the list cards and the post page header (already partially implemented).

### Claude's Discretion
- Exact breakpoint for desktop sidebar TOC vs mobile collapsible (default: sticky sidebar ≥1024px, collapsible below)
- TOC scroll offset and IntersectionObserver threshold (standard `-20%` rootMargin pattern)
- Share button visual treatment (icon-only or icon+label)
- Breadcrumb separator style and typography
- Cover image max-height and aspect ratio on hero post vs post page
- Tag pill style — reuse existing tag chip pattern or create new variant
- Series banner design (subtle alert-style bar or inline text)
- Transition/animation on filter tag selection (if any)

</decisions>

<specifics>
## Specific Ideas

- **Cover gradient fallback:** Reuse the cyan→violet `--accent` / `--accent-secondary` gradient tokens from the design system for cover image fallback — consistent with the hero section's visual treatment.
- **TOC auto-generation:** Parse headings from MDX content on the server side (in `getPostBySlug` or a TOC utility). Store as a `headings: { id, text, level }[]` field on the post meta object.
- **Series navigation:** Infer series from `post.series` field. Get all posts with the same series value, sorted by date, and render "Part N of M: Title" with prev/next links.
- **Share links:** Use the post's canonical URL (from frontmatter) or construct from base path. X share via `https://twitter.com/intent/tweet?text=...&url=...`, LinkedIn via `https://linkedin.com/sharing/share-offsite/?url=...`.
- **Breadcrumb generation:** Use the first tag as breadcrumb category (e.g., "Blog > Introduction > Hello World") or fall back to "Blog > Posts > Title" when no tags exist.
- **Hero/grid breakpoint:** Hero card spans full width, grid below at 2 columns. On mobile, grid collapses to 1 column and hero becomes more compact.
</specifics>
