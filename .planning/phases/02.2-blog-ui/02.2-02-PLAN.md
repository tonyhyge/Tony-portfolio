<task type="auto">
  <name>Task 1: Extract headings from MDX server-side</name>
  <files>
    src/app/data/blog.ts
  </files>
  <action>
    Add a headings extraction utility to the blog data module. This runs at build time during getPostBySlug.

    **Goal:** Add a function that parses `post.content` (raw MDX string) for h2 and h3 headings, returning an array of `{ id: string, text: string, level: number }`.

    **Implementation:**
    - Create a function `extractHeadings(content: string): { id: string; text: string; level: number }[]`
    - Use a simple regex: `/^(#{2,3})\s+(.+)$/gm` to match markdown headings (## and ###)
    - For each match:
      - `level`: `match[1].length` (2 or 3)
      - `text`: `match[2].trim()` — the raw heading text
      - `id`: generate a slug from the heading text: lowercase, replace non-alphanumeric with hyphens, collapse multiple hyphens, trim leading/trailing hyphens — same logic as what rehype-slug would generate
    - Call `extractHeadings(post.content)` inside `getPostBySlug()` and attach as `headings` field on the return object

    **Update BlogPostMeta type:**
    - Add `headings: { id: string; text: string; level: number }[]` to the type
    - Default to empty array if no headings found

    **Constraint:** Keep this server-side only (fs module, no client import). The regex approach is simpler and faster than pulling in rehype-slug which would add a build dependency and complexity for this scope.

    **Edge cases:**
    - Indented headings: `  ## Title` — the regex `^` anchor should match after optional whitespace
    - Headings with inline code: `## Using the `map` function` — extract "Using the `map` function" as text
    - Headings with links: `## [Link text](url)` — extract "Link text" as text
    - No headings: return empty array
    - Consecutive headings: all returned in order
    - Content with frontmatter: ensure frontmatter `---` delimiters are NOT processed (the content passed to extractHeadings is already frontmatter-stripped by gray-matter)
  </action>

  </verify>
