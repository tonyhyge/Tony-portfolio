# Phase 2.1 Plan 02: Blog Data Model Summary

Created the validated blog post frontmatter schema and build-time data access layer, following the same Zod + gray-matter pattern established in Phase 1.5 Research data model.

## Files Created

### `src/app/data/blog.ts` (57 lines)

- **BlogPostSchema** — Zod schema with 9 frontmatter fields (title, date, tags, excerpt, cover, draft, updated, series, canonical). Required fields validated via `z.string().min(1)` and regex; optional fields with `.optional()` or `.default()`.
- **BlogPost** — inferred type from schema
- **BlogPostMeta** — BlogPost extended with `slug: string` and `readingTime: number`
- **getAllSlugs()** — reads `content/blog/*.mdx` filenames via `readdirSync`, strips `.mdx` extension. Includes drafts.
- **getPostBySlug()** — reads file, parses frontmatter with gray-matter, validates via `BlogPostSchema.parse()`, computes word count and reading time
- **getAllPosts()** — maps slugs, filters out drafts, sorts by date descending
- **getAllTags()** — flat-maps tags from non-draft posts into sorted unique array

### `src/lib/blog.ts` (22 lines)

- **estimateReadingTime(content)** — splits on whitespace, returns `Math.ceil(wordCount / 200)`. Handles empty string (returns 0).
- **formatPostDate(dateStr)** — formats ISO date `2026-06-22` to `June 22, 2026`. Separate from `formatDate` in utils.ts which handles month/year format.

## Dependencies

- **gray-matter@4.0.3** — added for MDX frontmatter parsing. Ships with bundled `gray-matter.d.ts` type declarations; no `@types/gray-matter` needed.
