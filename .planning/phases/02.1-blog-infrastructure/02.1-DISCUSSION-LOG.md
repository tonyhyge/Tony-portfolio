## MDX Parsing Approach

| Option | Description | Selected |
|--------|-------------|----------|
| next-mdx-remote | Serialize MDX at build time, render with `<MDXRemote>`. Works with dynamic content/ dir, supports embedded React components, proven with static export. | ✓ |
| rehype/remark only | Pure unified pipeline — parse MDX to HTML, no React component embedding in posts. Lighter but no custom blog components. | |

**User's choice:** next-mdx-remote
**Notes:** Standard choice for Next.js static export blog. Supports embedded React components for rich blog posts.

## Frontmatter Schema & Data Model

| Option | Description | Selected |
|--------|-------------|----------|
| Minimal set | Only title, date, tags, excerpt, cover. Slug from filename. No drafts, series, or canonical URL. | |
| Extended set | Add: draft, updated, series, canonical. Slug from filename. | ✓ |

**User's choice:** Extended set
**Notes:** Includes draft (pre-publish hiding), updated (last edit date), series (multi-part posts), canonical (cross-post attribution).

## Data Validation Approach

| Option | Description | Selected |
|--------|-------------|----------|
| Zod schema + gray-matter | Parse frontmatter with gray-matter, validate with Zod. Typed, validated frontmatter at build time. | ✓ |
| TypeScript interface only | Just type the frontmatter interface — no runtime validation. Simpler, less boilerplate. | |

**User's choice:** Zod schema + gray-matter
**Notes:** Same pattern as Research data model (Phase 1.5). Consistency across the codebase.

## Content Directory & Routing

| Option | Description | Selected |
|--------|-------------|----------|
| Dynamic route: /blog/[slug] | src/app/blog/[slug]/page.tsx with generateStaticParams(). Clean URLs. | ✓ |
| Flat listing only | Single /blog page, all posts there. No individual post routes. | |

**User's choice:** Dynamic route: /blog/[slug]
**Notes:** Standard App Router pattern. Shareable URLs per post. Phase 2.2 builds the list and post pages.

## Syntax Highlighting Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| rehype-pretty-code | Built on shiki — zero runtime JS, dual dark/light themes, line highlighting. | ✓ |
| rehype-highlight | Uses highlight.js. Simpler, smaller, fewer features. | |

**User's choice:** rehype-pretty-code
**Notes:** The modern standard for Next.js MDX blogs. Zero runtime JS in browser, themes at build time.

## Reading Time Estimation

| Option | Description | Selected |
|--------|-------------|----------|
| Word count only | Math.ceil(wordCount / 200) min. Simple, standard. | ✓ |
| Word count + image adjustment | Base formula + ~12s per image. More accurate for image-heavy posts. | |

**User's choice:** Word count only
**Notes:** Keeps it simple. Computed at build time from MDX source before rendering.
