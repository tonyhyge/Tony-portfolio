  <action>
    Create `src/app/blog/[slug]/page.tsx` as an async Server Component (no `"use client"` directive).

    **Imports:**
    - `getAllSlugs, getPostBySlug` from `@/app/data/blog`
    - `type BlogPostMeta` from `@/app/data/blog`
    - `serialize` from `next-mdx-remote/serialize`
    - `MDXRemote` from `next-mdx-remote`
    - `rehypePrettyCode` from `rehype-pretty-code`
    - `notFound` from `next/navigation`

    **`generateStaticParams()`** — per D-06, D-07:
    - Call `getAllSlugs()` — this includes draft slugs per Plan 02's discretion decision (drafts accessible by URL)
    - Return `{ slug }[]` array

    **`Page` component:**
    - Accept `{ params: { slug: string } }` — note: Next.js 16 uses unwrapped `params` (no async/Promise needed)
    - Call `getPostBySlug(slug)` to get post data
    - If post not found or file missing, call `notFound()`
    - Serialize MDX content with `serialize(post.content, { mdxOptions: { rehypePlugins: [] } })`
    - Wrap serialization in try-catch; on error throw with descriptive message (MDX parse errors surface at build time)

    **Syntax highlighting — rehype-pretty-code config per D-08 and discretion resolution:**
    ```typescript
    rehypePrettyCode, {
      theme: {
        dark: "github-dark-dimmed",
        light: "github-light",
      },
      keepBackground: false,
      defaultLang: "plaintext",
    }
    ```
    Decision rationale: "github-dark-dimmed" (not full "github-dark") produces lower visual contrast that pairs well with the portfolio's oklch(0.145) dark background. "github-light" is a clean, familiar light theme. `keepBackground: false` lets the site's own background color show through, respecting the `.dark` class toggle. `defaultLang: "plaintext"` prevents unlabeled code blocks from failing silently.

    **Render structure** (basic — Phase 2.2 handles visual polish):
    ```tsx
    <article className="mx-auto max-w-3xl px-4 py-12">
      <header className="mb-8">
        <h1 className="text-3xl font-semibold tracking-tight">{post.title}</h1>
        <div className="mt-2 flex items-center gap-4 text-sm text-muted-foreground">
          <time dateTime={post.date}>{formatPostDate(post.date)}</time>
          <span aria-label={`${post.readingTime} minute read`}>{post.readingTime} min read</span>
        </div>
        {post.tags.length > 0 && (
          <div className="mt-4 flex flex-wrap gap-2">
            {post.tags.map((tag) => (
              <span key={tag} className="rounded-full bg-muted px-3 py-1 text-xs font-medium text-muted-foreground">
                {tag}
              </span>
            ))}
          </div>
        )}
      </header>
      <div className="prose prose-neutral dark:prose-invert max-w-none">
        <MDXRemote source={mdxSource} />
      </div>
    </article>
    ```

    Import `formatPostDate` from `@/lib/blog`. The prose classes (`prose prose-neutral dark:prose-invert`) provide base typography for rendered MDX content. The `max-w-none` override stretches prose to fill the article container instead of capping at 65ch.

    For `article` element: add explicit `lang="en"` attribute for accessibility.

    Do NOT add any "use client" imports or interactive elements — the rendering is fully static.
