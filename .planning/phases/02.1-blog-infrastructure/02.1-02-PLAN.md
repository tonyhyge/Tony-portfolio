<task type="auto">
  <name>Task 1: Create BlogPostSchema, BlogPost type, and build-time data loaders</name>
  <files>src/app/data/blog.ts</files>
  <action>
    Create `src/app/data/blog.ts` following the exact pattern from `src/app/data/research.ts`:

    1. **Zod imports & schema** — per D-02, D-04:
       - Import `z` from "zod", `readFileSync`/`readdirSync` from "fs", `join` from "path", `matter` from "gray-matter"
       - Export `BlogPostSchema` as `z.object({...})` with:
         - `title`: z.string().min(1) — required
         - `date`: z.string().regex(/^\d{4}-\d{2}-\d{2}$/) — required, ISO date
         - `tags`: z.array(z.string()).default([]) — optional, defaults to empty
         - `excerpt`: z.string().optional()
         - `cover`: z.string().optional()
         - `draft`: z.boolean().default(false) — optional, defaults to false
         - `updated`: z.string().regex(/^\d{4}-\d{2}-\d{2}$/).optional()
         - `series`: z.string().optional()
         - `canonical`: z.string().url().optional()
       - Export `BlogPost` = `z.infer<typeof BlogPostSchema>`
       - Export `BlogPostMeta` = `BlogPost & { slug: string; readingTime: number }`

    2. **Directory constant**:
       - `POSTS_DIR = join(process.cwd(), "content", "blog")`

    3. **`getAllSlugs()`**:
       - `readdirSync(POSTS_DIR)`, filter `.mdx`, map to slug (strip `.mdx`)
       - **Draft handling per discretion decision:** Include draft slugs so drafts are accessible by direct URL but filtered from listing

    4. **`getPostBySlug(slug)`**:
       - `readFileSync` the .mdx file
       - Parse with `matter(source)` — returns `{ data, content }`
       - Validate `data` via `BlogPostSchema.parse(data)` — throws on invalid frontmatter
       - Compute word count from `content` via `content.split(/\s+/g).filter(Boolean).length`
       - Compute readingTime = `Math.ceil(wordCount / 200)` per D-09
       - Return `{ ...parsed, slug, readingTime }`

    5. **`getAllPosts()`**:
       - `getAllSlugs().map(getPostBySlug)`
       - Filter out drafts: `.filter(p => !p.draft)`
       - Sort by date descending: `.sort((a, b) => new Date(b.date).getTime() - new Date(a.date).getTime())`

    6. **`getAllTags()`**:
       - Flat-map all posts' tags into a Set, return sorted array
       - Only non-draft posts contribute to the tag list

    The PATTERNS.md file (lines 32-91) contains the exact code structure — follow that closely.

    Use `node:crypto` style imports from `node:*` for fs/path.
