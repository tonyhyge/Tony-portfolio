# Phase 2.1: Blog Infrastructure — Context

**Gathered:** 2026-06-22
**Status:** Ready for planning

<domain>
## Phase Boundary

MDX-based blog engine foundation — data schema, content directory structure, MDX parsing pipeline, syntax highlighting, reading time estimation. The backend/infrastructure layer that Phase 2.2 (Blog UI) and Phase 2.3 (Blog Features) build on.

**In scope:** MDX integration (next-mdx-remote), blog post frontmatter schema (Zod), content directory structure (`/content/blog/*.mdx`), dynamic route (`/blog/[slug]`), syntax highlighting via rehype-pretty-code, reading time estimation.
**Not in scope:** Blog list page UI (Phase 2.2), individual post page rendering (Phase 2.2), tag filtering UI (Phase 2.2), RSS feed (Phase 2.3), SEO per post (Phase 2.3), share buttons (Phase 2.3), pagination (Phase 2.3).
</domain>

<decisions>
## Implementation Decisions

### MDX Parsing Approach
- **D-01:** Use **next-mdx-remote** — serialize MDX at build time, render with `<MDXRemote>`. Works with dynamic `content/` directory, supports embedded React components, proven with static export. Community standard for Next.js portfolios.

### Frontmatter Schema
- **D-02:** Extended frontmatter fields:
  - `title` (required) — string
  - `date` (required) — ISO date string (YYYY-MM-DD)
  - `tags` (optional) — string[]
  - `excerpt` (optional) — string (short summary for cards)
  - `cover` (optional) — cover image path
  - `draft` (optional) — boolean (hide from listing, default false)
  - `updated` (optional) — ISO date string (last edited date)
  - `series` (optional) — string (for multi-part posts)
  - `canonical` (optional) — URL string (for cross-posts)
- **D-03:** Slug inferred from filename (e.g., `my-first-post.mdx` → `/blog/my-first-post`)
- **D-04:** Frontmatter validated with **Zod schema** + gray-matter parser at build time — same pattern as Research data model (Phase 1.5).

### Content Directory & Routing
- **D-05:** Posts stored as `content/blog/<slug>.mdx` files
- **D-06:** Dynamic route: `src/app/blog/[slug]/page.tsx` with `generateStaticParams()` reading `content/blog/` directory at build time
- **D-07:** Clean URLs: `/blog/my-first-post`

### Syntax Highlighting
- **D-08:** **rehype-pretty-code** — rehype plugin built on shiki. Zero runtime JS, dual dark/light theme support, line highlighting, word highlighting. Runs at build time, no browser-side highlighter.

### Reading Time Estimation
- **D-09:** Simple word-count formula: `Math.ceil(wordCount / 200)` → "X min read"
- **D-10:** Computed at build time from raw MDX source before rendering — no runtime JS

### Claude's Discretion
- Exact rehype-pretty-code theme selection for dark/light mode
- Directory structure for blog utility functions (`src/lib/blog.ts` or similar)
- Whether to extract a shared `_components/mdx.tsx` for MDX component mapping
- Draft filtering implementation (exclude from list, still accessible via direct URL or hidden entirely)
- Series grouping data model (how series name maps to related posts)

</decisions>
