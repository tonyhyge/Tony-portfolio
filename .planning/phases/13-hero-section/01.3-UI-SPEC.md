## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing (between name/tagline/CTAs) |
| lg | 24px | Section padding (container edges) |
| xl | 32px | Layout gaps (split layout text‚Üîhexagon) |
| 2xl | 48px | Major section breaks (CTA group to scroll indicator) |
| 3xl | 64px | Page-level spacing |

**Exceptions:**
- Hexagon badge: 120px desktop / 64px mobile (D-07, D-17 in CONTEXT.md) ‚Äî not on spacing scale, geometric shape dimension.
- Hero section height: `min-h-[90vh]` (D-02 in CONTEXT.md) ‚Äî viewport-relative, not additive spacing scale.

**Source:** Default 8-point scale from template, confirmed as reasonable defaults. Existing Tailwind v4 utility classes (`gap-4`, `gap-6`, etc.) map directly.

2. Hexagon badge border/glow effect
3. Primary CTA button background (via `--primary` which maps to accent tones in dark theme)
4. Primary CTA focus-visible ring
5. "Download CV" text link color

**Not accent:** Tagline text (uses `--foreground`), scroll indicator text (uses `--muted-foreground`), outline CTA border (uses `--border`).

**Source:** Pre-populated from `globals.css` `:root` and `.dark` tokens. Accent colors confirmed in CONTEXT.md D-14 (cyan‚Üíviolet gradient).

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Hero on load | Content visible immediately, no loading state. Background animating (20s cycle via `animate-gradient-shift`). |
| Scroll-down indicator on load | Fully visible (opacity-100), gentle pulse animation on arrow (CSS `transform: translateY(4px)` cycle, 2s). |
| Scroll-down indicator on scroll | Fades to opacity-0 within first 100‚Äì200px scroll distance. CSS `transition: opacity 300ms ease-out`. No JS timer. |
| "View Projects" click | Smooth scroll to `#experience` section (next phase content). |
| "Contact Me" click | Smooth scroll to `#contact` section. |
| "Download CV" click | Opens placeholder URL in new tab (`target="_blank" rel="noopener noreferrer"`). |
| Reduced motion | `motion-reduce` disables background animation + scroll indicator pulse. Scroll fade still operates (functional, not decorative). |
| Keyboard focus | Primary CTA gets `focus-visible:ring-3 focus-visible:ring-ring/50` from Button component. All interactive elements tabbable. |
| Responsive (< 768px) | Split layout ‚Üí stacked layout. Hexagon shrinks to 64px, stays inline right of text block (not above, not hidden). |

**Source:** Pre-populated from CONTEXT.md D-11 (fade behavior), D-12 (CSS opacity transition), and project NFR2 (accessibility) + NFR3 (responsiveness).

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<section id="home" aria-label="Hero">` wrapper. Name in `<h1>`, tagline in `<p>`. |
| Color contrast | Name gradient text: verify both cyan and violet endpoints meet 4.5:1 against dark background (oklch(0.145 0 0)). Tagline uses `--foreground` (0.985 oklch on 0.145 = pass). |
| Keyboard navigation | CTAs are native `<button>` elements, tabbable. Focus indicators via shadcn Button `focus-visible` styles. |
| Reduced motion | Background respects `motion-reduce:animate-none`. Scroll indicator pulse disabled. Fade-on-scroll retained (functional). |
| Screen reader | `HeroBackground` has `aria-hidden="true"`. Hexagon badge has `aria-hidden="true"` (decorative). Name/tagline are plain text ‚Äî no ARIA override needed. |
| Touch targets | CTAs at `lg` size (min 36px height) meet 44px touch target when combined with padding. |

**Source:** NFR2 (WCAG AA compliance). Pre-populated from existing `HeroBackground` `aria-hidden="true"` and shadcn Button focus-visible pattern.
