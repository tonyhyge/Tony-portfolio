## Summary

This phase implements two sections on the single-page portfolio: a Skills section with categorized accordion tags and cross-category project linking, plus a Contact section with Formspree form backend and social links row. Both sections are client components with distinct visual treatments.

**Primary recommendation:** Use `@formspree/react` v3.0.0 for the contact form (MIT-licensed, `useForm()` hook handles submission/validation/confirmation states natively). Build a lightweight `useClickOutside` hook for the custom popover. Extend the existing CVA badge variant system for per-category skill tag colors.

### Pattern 1: Accordion with scrollHeight Animation
**What:** Smooth expand/collapse using JS-calculated `scrollHeight` mapped to CSS `max-height` transition. Only one category open at a time.
**When to use:** Controlled accordion where you need smooth height animation (not instant toggle).
**Example:**
```tsx
function AccordionContent({ isOpen }: { isOpen: boolean }) {
  const ref = useRef<HTMLDivElement>(null)
  const [height, setHeight] = useState(0)

  useEffect(() => {
    if (ref.current) {
      setHeight(isOpen ? ref.current.scrollHeight : 0)
    }
  }, [isOpen])

  return (
    <div
      ref={ref}
      style={{ maxHeight: height }}
      className="overflow-hidden transition-[max-height] duration-300 ease-out"
    >
      <div className="py-2">{/* skill tags */}</div>
    </div>
  )
}
```

### Pattern 2: Custom Click-Outside Hook
**What:** Detect clicks outside a referenced element to close popovers/menus.
**When to use:** Any custom dropdown, popover, or menu that needs outside-click dismissal.
**Example:**
```tsx
function useClickOutside(handler: () => void) {
  const ref = useRef<HTMLDivElement>(null)

  useEffect(() => {
    function listener(e: MouseEvent | TouchEvent) {
      if (!ref.current || ref.current.contains(e.target as Node)) return
      handler()
    }
    document.addEventListener("mousedown", listener)
    document.addEventListener("touchstart", listener)
    return () => {
      document.removeEventListener("mousedown", listener)
      document.removeEventListener("touchstart", listener)
    }
  }, [handler])

  return ref
}
```

### Pattern 3: Formspree useForm Integration
**What:** React hook wrapping Formspree form submission with typed validation errors, loading, and success states.
**When to use:** Any form that sends data to Formspree with React UX states (no full page reload).
**Example:**
```tsx
import { useForm, ValidationError } from "@formspree/react"

export function ContactForm() {
  const [state, handleSubmit] = useForm("YOUR_FORM_ID")

  if (state.succeeded) {
    return <p>Thanks, Minh will get back to you soon.</p>
  }

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="name">Name</label>
      <input id="name" type="text" name="name" required />
      <ValidationError field="name" errors={state.errors} />

      <label htmlFor="email">Email</label>
      <input id="email" type="email" name="email" required />
      <ValidationError prefix="Email" field="email" errors={state.errors} />

      <label htmlFor="message">Message</label>
      <textarea id="message" name="message" required />
      <ValidationError prefix="Message" field="message" errors={state.errors} />

      {/* Honeypot — D-15 */}
      <input type="text" name="_gotcha" className="hidden" tabIndex={-1} autoComplete="off" />

      <button type="submit" disabled={state.submitting}>Send</button>
    </form>
  )
}
```

### Anti-Patterns to Avoid
- **shadcn Sheet for skill linking:** D-09 explicitly rules this out — Sheet is for mobile nav, not tag-adjacent popovers.
- **Progress bars for proficiency:** D-01/D-02 forbid self-assessed progress bars. Proficiency is proven via Experience/Research sections.

- **Layout-animating properties:** Animate `max-height` via CSS transition, not `height` directly. Follow web/coding-style.md: prefer compositor-friendly motion.

## Common Pitfalls

### Pitfall 1: Formspree Form ID Not Configured
**What goes wrong:** `useForm("YOUR_FORM_ID")` with placeholder ID causes 404 on submission.
**Why it happens:** Formspree requires creating a form in their dashboard first to get the unique form hash ID.
**How to avoid:** Create Formspree account → create form → copy form ID → use as `useForm("{form-id}")`. Consider `process.env.NEXT_PUBLIC_FORMSPREE_ID` for environment variable storage.
**Warning signs:** Console error on form submit: "POST https://formspree.io/f/{id} 404 Not Found".

### Pitfall 2: Accordion Height Animation Jank
**What goes wrong:** `max-height` set to a large fixed value (e.g., `max-h-96`) causes animation duration mismatch — short content animates too fast, tall content gets clipped.
**Why it happens:** CSS `transition` animates to the specified `max-height` value, not the actual content height.
**How to avoid:** Use JS-calculated `scrollHeight` for the exact `max-height` value per D-04. The pattern above (Pattern 1) solves this correctly.
**Warning signs:** Animation feels instant for short categories or cuts off for long ones.

### Pitfall 3: Popover Z-Index Stacking
**What goes wrong:** Custom popover renders behind other content or gets clipped by overflow containers.
**Why it happens:** Missing `z-index` management or parent has `overflow: hidden`.
**How to avoid:** Render popover via React Portal to `document.body`, or ensure parent containers don't clip. Use `z-50` or higher. Set `position: fixed` for reliable viewport-relative positioning.
**Warning signs:** Popover partially invisible or completely hidden behind adjacent sections.

### Pitfall 4: Cross-Category Skill Data Duplication
**What goes wrong:** Same skill (e.g., "PyTorch") appears in multiple categories but is stored as separate entries, making updates error-prone.
**Why it happens:** Old data model uses `string[]` per category — no single source of truth.
**How to avoid:** New `Skill` model (D-07) with `categories: string[]` array. Each skill is a single object that declares which categories it belongs to. Build category-filtered views at render time.
**Warning signs:** Editing a skill requires updating multiple array entries.

### Pitfall 5: Honeypot Field Accessibility
**What goes wrong:** Honeypot field accidentally visible to screen readers or keyboard users, blocking legitimate submissions.
**Why it happens:** Using `hidden` attribute or `display: none` doesn't fully hide from assistive tech.
**How to avoid:** Use `tabIndex={-1}`, `autoComplete="off"`, `aria-hidden="true"`, and CSS `position: absolute; left: -9999px` instead of `display: none`.
**Warning signs:** Legitimate form submissions rejected as spam.
