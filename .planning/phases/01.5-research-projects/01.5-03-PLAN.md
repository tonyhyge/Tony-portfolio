  <action>
    Create src/components/research/ResearchGrid.tsx:

    **Structure: Two components in one file: `ResearchGrid` (outer — exports, Suspense wrapper) and `ResearchGridInner` (inner — uses useSearchParams).**

    ------
    `ResearchGrid` (outer, default export of the file):
    
    - No `"use client"` directive (this is a Server Component wrapper).
    
    - Import { Suspense } from "react".
    - Import { researchProjects, getAvailableStatuses } from "@/app/data/research".
    - Import dynamic wrapper for the inner component OR just render it inside Suspense.

    - `export function ResearchGrid()`:
      ```
      return (
        <Suspense fallback={<div className="grid grid-cols-1 gap-6 md:grid-cols-2">{/* Minimal skeleton placeholder. Use a min-height container to avoid layout shift */}</div>}>
          <ResearchGridInner projects={researchProjects} />
        </Suspense>
      );
      ```
    
    - Fallback: simple `<div className="grid grid-cols-1 gap-6 pt-4 md:grid-cols-2"><div className="h-48 md:col-span-2 rounded-xl bg-muted/30 animate-pulse" /></div>` — a subtle skeleton placeholder matching the grid structure.

    ------
    `ResearchGridInner` (inner function, NOT exported):

    - `"use client"` directive at top of file (before the ResearchGrid export, after imports).

    - Import { useState, useEffect, useCallback, useMemo } from "react".
    - Import { useSearchParams, useRouter, usePathname } from "next/navigation".
    - Import { FilterTabs } from "./FilterTabs".
    - Import { ResearchCard } from "./ResearchCard".
    - Import { Button } from "@/components/ui/button".
    - Import { cn } from "@/lib/utils".

    - `interface ResearchGridInnerProps { projects: ResearchProject[] }`

    - `function ResearchGridInner({ projects }: ResearchGridInnerProps)`:
      - `const searchParams = useSearchParams()`
      - `const router = useRouter()`
      - `const pathname = usePathname()`

      - Active status from URL (memoized):
        ```typescript
        const activeStatus = useMemo(() => searchParams.get("status") || "all", [searchParams]);
        ```

      - Filtered projects (memoized):
        ```typescript
        const filteredProjects = useMemo(() => {
          if (activeStatus === "all") return projects;
          return projects.filter((p) => p.status === activeStatus);
        }, [projects, activeStatus]);
        ```

      - Available statuses (memoized):
        ```typescript
        const availableStatuses = useMemo(() => getAvailableStatuses(), []);
        ```
        (getAvailableStatuses reads from the static import, it doesn't need dependencies.)

      - Transition state for crossfade (D-11):
        ```typescript
        const [transitioning, setTransitioning] = useState(false);
        ```

      - Filter change handler:
        ```typescript
        const handleFilterChange = useCallback((status: string) => {
          setTransitioning(true);
          const params = new URLSearchParams(searchParams.toString());
          if (status === "all") params.delete("status");
          else params.set("status", status);
          
          // Use setTimeout to allow exit transition to play before navigating
          // then navigation triggers re-render with new searchParams
          setTimeout(() => {
            router.push(pathname + "?" + params.toString());
            // Transition resets when new searchParams arrive
          }, 150);
        }, [router, pathname, searchParams]);
        ```

      - Reset filter handler (for empty state "Show all"):
        ```typescript
        const handleShowAll = useCallback(() => {
          router.push(pathname);
        }, [router, pathname]);
        ```

      - Crossfade effect: when transitioning becomes true after a status filter change, the grid fades out. Then when the URL update triggers a re-render with new searchParams, setTransitioning(false) must fire. Use a clean-up approach:
        
        Track prevActiveStatus via a ref. When activeStatus changes, set transitioning(false). This naturally occurs on the re-render after router.push.
        
        ```typescript
        const prevStatusRef = useRef(activeStatus);
        
        useEffect(() => {
          // Status changed via router.push — end transition
          if (prevStatusRef.current !== activeStatus) {
            prevStatusRef.current = activeStatus;
            setTransitioning(false);
          }
        }, [activeStatus]);
        ```

      - Browser check for reduced motion:
        ```typescript
        const prefersReducedMotion = typeof window !== "undefined"
          ? window.matchMedia("(prefers-reduced-motion: reduce)").matches
          : false;
        
        const isTransitioning = transitioning && !prefersReducedMotion;
        ```

      - JSX render:
        ```
        <div className="space-y-8">
          <FilterTabs
            availableStatuses={availableStatuses}
            activeStatus={activeStatus}
            onFilterChange={handleFilterChange}
          />
          
          <div
            className={cn(
              "grid grid-cols-1 gap-6 md:grid-cols-2",
              isTransitioning ? "opacity-0 transition-all duration-150 ease-out" : "opacity-100 transition-all duration-200 ease-out",
            )}
          >
            {filteredProjects.length > 0 ? (
              filteredProjects.map((project) => (
                <ResearchCard
                  key={project.title}
                  project={project}
                />
              ))
            ) : (
              /* Empty state — shown when filtered set is empty */
              <div className="col-span-1 md:col-span-2 flex flex-col items-center justify-center py-16 text-center">
                <p className="text-lg font-medium text-foreground">
                  No {formatStatusLabel(activeStatus)} projects yet
                </p>
                <p className="mt-1 text-sm text-muted-foreground">
                  Check back soon for updates to this section.
                </p>
                {activeStatus !== "all" && (
                  <Button
                    variant="outline"
                    className="mt-4"
                    onClick={handleShowAll}
                  >
                    Show all
                  </Button>
                )}
              </div>
            )}
          </div>
        </div>
        ```

      - formatStatusLabel helper (same as Plan 02, defined inside file):
        ```typescript
        function formatStatusLabel(status: string): string {
          const labels: Record<string, string> = {
            "in-progress": "In Progress",
            submitted: "Submitted",
            published: "Published",
            accepted: "Accepted",
          };
          return labels[status] ?? status;
        }
        ```

    **Important:** The ResearchGridInner component uses useSearchParams which requires a Suspense boundary. The parent ResearchGrid component provides this. Do NOT remove the Suspense boundary — static export build fails without it.

    **Crossfade mechanics:**
    - When filter changes: transitioning=true (150ms) → grid fades out → URL updates → new searchParams cause re-render → transitioning=false → grid fades in (200ms)
    - No scale transform (avoids clipping on col-span-2 featured card — per RESEARCH.md Pitfall 4)
    - `transition-all duration-150 ease-out` for exit, `transition-all duration-200 ease-out` for enter
    - prefers-reduced-motion check disables the transition entirely (instant switch)

    Export only `ResearchGrid` from the file — `ResearchGridInner` is internal.
