## Spacing Scale

Tailwind v4 default scale (`--spacing-*` tokens). Deviations listed below:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, paper link chip inner padding |
| sm | 8px | Badge-to-text gap, chip gap |
| md | 16px | Card padding, grid gap |
| lg | 24px | Section padding (container edges), filter bar margin-bottom |
| xl | 32px | Card-to-card gap, featured card extra padding |
| 2xl | 48px | Major section breaks (filter bar to card grid) |
| 3xl | 64px | Section top margin (from preceding section) |

**Exceptions:**
- Bento grid gap: 24px (`gap-6`) — matches `lg` for visual breathing room between cards of different sizes.
- Featured card vertical emphasis: 8px extra `pt-2` top padding (keeps content aligned to card top without uniform spacing).
- Abstract truncated text: `line-clamp-3` (3 lines, not a spacing token).

**Source:** CONTEXT.md D-01 (bento grid 2-col desktop), D-02 (featured 2-col). Spacing scale inherited from Phase 1.1.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (`text-base`) | 400 | 1.5 | Abstract text, filter tab label, venue name, collaboration credit |
| Label | 14px (`text-sm`) | 400 | 1.4 | Status badge text, paper link label, date |
| Heading | 20px (`text-xl` featured, `text-lg` non-featured) | 700 (bold) | 1.3 | Project card titles — featured uses larger size for hierarchy |
| Section heading | 28px (`text-3xl` on mobile, `text-4xl` desktop) | 700 | 1.1 | "Research" section title in page |

**Featured vs non-featured distinction:** Featured card title at 20px creates hierarchy without adding a 5th font size — `text-lg` (18px) and `text-xl` (20px) are both in the "heading" role, same weight. The featured card's primary visual distinction comes from layout (2-column bento span) and accent border-top, not a separate typographic role.

**Section heading** inherits page heading style (consistent with "About" and "Experience" from Phase 1.4).

**Abstract expand toggle:** "Show more" / "Show less" link at `text-sm` weight 500, accent color, no icon (text-only toggle keeps card compact).

**Section heading** inherits page heading style (consistent with "About" and "Experience" from Phase 1.4).

**Source:** CONTEXT.md D-15 (abstract truncated), D-16 (expand/collapse). Sizes chosen for card readability at 2-col grid width (~400-500px per card).

### Status Badge Colors (D-05)

| Status | Color | oklch Value | Dark Variant |
|--------|-------|-------------|--------------|
| `in-progress` | Amber/warm | `oklch(0.72 0.15 72)` | `oklch(0.72 0.12 72)` |
| `submitted` | Blue | `oklch(0.55 0.14 250)` | `oklch(0.6 0.12 250)` |
| `published` | Green | `oklch(0.58 0.17 145)` | `oklch(0.62 0.14 145)` |
| `accepted` | Purple | `oklch(0.52 0.16 290)` | `oklch(0.58 0.14 290)` |

Badge background uses 15% opacity of the status color (bg with low alpha). Text uses full opacity status color.

**Accent reserved for (this phase):**
1. Featured card top border accent line (thin 2px `--accent` border-top, or subtle gradient overlay)
2. Paper link chip hover state (background tint)
3. "Show more" / "Show less" abstract toggle text
4. Filter tab active indicator (underline or background tint in `--accent`)
5. Status badge dot indicator (small circle before status label)

**Not accent:** Card title text (uses `--foreground`), abstract text (uses `--muted-foreground`), filter tab text (uses `--foreground` for active, `--muted-foreground` for inactive), card background (uses `--card`).

**Source:** CONTEXT.md D-05 (status badge colors per status), globals.css project tokens.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section heading | "Research" |
| Section subheading | "Published & ongoing work in AI and ML" |
| Empty state heading | "No {status} projects yet" |
| Empty state body | "Check back soon for updates to this section." + "Show all" button |
| Filter "All" tab | "All" |
| Filter status tabs | "In Progress" / "Submitted" / "Published" / "Accepted" |
| Abstract expand | "Show more" / "Show less" |
| Paper link label | "Paper" (for DOI/arXiv), "PDF" (for direct PDF) |
| No paper link fallback | Not shown — chip only renders when link exists |
| Section CTA | None — inline filter IS the interactive control |
| Featured card | No special copy, just visual emphasis via layout + border accent |

**Empty state rationale:** When filter selects a status with zero results, show "No {status} projects yet" + "Show all" button that resets `?status=` param to "all". Consistent with the auto-detect tab behavior (D-10) — if a status tab has zero total projects, the tab itself is hidden; empty state only triggers when user navigates to a tab that existed but has no data at that moment (e.g., after future content changes).

**Source:** CONTEXT.md D-09 (filter tabs), D-10 (auto-detect tabs), D-17 (empty state message + button).

## Component Architecture

| Component | Path | Purpose |
|-----------|------|---------|
| `ResearchGrid` | `src/components/research/ResearchGrid.tsx` | Main section — heading, filter bar, bento card grid, empty state. Wires URL search params for filter state. |
| `ResearchCard` | `src/components/research/ResearchCard.tsx` | Individual project card — title, status badge, abstract (truncated + expand), tags, paper link chips, featured treatment. |
| `FilterTabs` (inline in ResearchGrid or separate) | `src/components/research/FilterTabs.tsx` | Row of filter chips/tabs. Auto-detects available statuses from data. Crossfade animation on change. |
| `PaperLinkChip` | `src/components/research/PaperLinkChip.tsx` | Small icon + label chip for arXiv/DOI/PDF links. Renders only for fields that have values. |

**Data model:** `src/data/research.ts` — typed array with Zod schema validation (D-12, D-13). See CONTEXT.md D-14 for full field list.

**Primary visual anchor:** The featured project card is the focal point of the research section. At 2-column bento span (vs 1-column for normal cards) with an accent top border and larger title, it draws the eye first on the grid. On mobile, hierarchy is maintained via accent border and title weight rather than column span.

**Section integration:** Rendered inside `<section id="research">` in `src/app/page.tsx`, positioned after the Experience section (per Phase 1.4 order). Follows same composition pattern as About and Experience.

**Source:** CONTEXT.md D-01 through D-17. Component breakdown follows feature-based organization per `web/coding-style.md`.

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Filter tab click | Updates URL `?status=` search param. Triggers crossfade animation on card grid. No page reload. |
| Crossfade animation | CSS transition: exiting cards fade out + scale(0.98), new cards fade in + scale(1). Duration ~200-250ms, ease-out. Grid container maintains min-height during transition. |
| Abstract expand | "Show more" click expands `line-clamp-3` → full text. "Show less" collapses back. Single toggle per card, independent state. |
| Paper link chip hover | Background tint (8% accent opacity), cursor pointer, no text decoration. Opens in new tab (`target="_blank" rel="noopener noreferrer"`). |
| Featured card hover | Subtle lift effect: `transform: translateY(-2px)`, `box-shadow` enhancement from card default. Duration 200ms, ease-out. |
| Non-featured card hover | Same lift effect as featured (consistent interaction language). |
| Card click | Entire card is clickable — navigates to `/research/<slug>` (for future detail pages). `cursor-pointer` on card. |
| Empty state | Shows message + "Show all" button. "Show all" resets `?status=` to show all projects. |
| Responsive (mobile, < 768px) | Bento grid collapses to 1 column. Featured card loses 2-col span but retains visual hierarchy (accent top border, larger title). Filter tabs horizontal-scroll if needed. |
| Reduced motion | Crossfade animation skips (instant switch). Card hover transform disabled. |

**Crossfade detail:** Use React state to track current + next filter. When filter changes:
1. Capture current cards' positions/visibility
2. Apply `opacity-0 scale-98` exit to outgoing set (150ms)
3. Swap data to new filter set
4. Apply `opacity-0 scale-98` initially, then `opacity-100 scale-100` enter (200ms)
5. Clean up transition classes

No animation library needed — pure CSS transitions + React state toggle.

**Source:** CONTEXT.md D-09 (URL param filter), D-11 (crossfade animation), D-15 (abstract truncated), DISCUSSION-LOG responsive decisions.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Semantic HTML | `<section id="research" aria-labelledby="research-heading">`. Card grid as `<div role="list">`, cards as `<div role="listitem">`. |
| Filter tabs | Native `<button>` elements with `aria-pressed` or `role="tablist"` + `role="tab"`. Active tab has `aria-selected="true"`. |
| Color contrast | Status badge colors: verify amber (#B8932A), blue (#3B82F6 on dark/light), green (#16A34A), purple (#9333EA) meet 4.5:1 against badge background. |
| Keyboard navigation | Filter tabs: arrow key navigation within tablist. Cards tabbable via single `<a>` or `<button>` wrapper. Paper link chips tabbable individually. |
| Reduced motion | Crossfade animation disabled (`prefers-reduced-motion`). Card hover transforms disabled. Filter switch becomes instant. |
| Screen reader | Status badge text includes both label + status (e.g., "Status: Published"). Abstract expand toggle announces expanded state via `aria-expanded`. Paper links announce "opens in new tab". |
| Touch targets | Filter tab min 44px height. Paper link chips min 36px height (meet 44px when padding included). |
| Focus indicators | `focus-visible:ring-2 focus-visible:ring-ring/50` on filter tabs, card wrappers, paper link chips. |

**Source:** CONTEXT.md NFR2 (WCAG AA compliance). Follows patterns established in Phase 1.3 UI-SPEC.
