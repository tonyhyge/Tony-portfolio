# Phase 01.5 Plan 03: Research Grid & Integration Summary

Created ResearchGrid (the composition orchestrator for the research section) and wired it into the home page. ResearchGrid coordinates URL-driven filter state, crossfade animation, empty state handling, and the Suspense boundary required by Next.js 16 static export. The research section now renders live after Experience on the home page.

## Tasks

### Task 1: ResearchGrid component — Complete (706d118)

ResearchGrid uses a two-component structure per Next.js 16 requirements: the outer `ResearchGrid` (exported, no `"use client"`) wraps `ResearchGridInner` (client component with `useSearchParams`) in a `<Suspense>` boundary. The fallback shows a skeleton placeholder matching the grid layout.

`ResearchGridInner` manages:
- **Filter state** via `useSearchParams` — reads `?status=` from URL, shareable via copy-paste
- **Filtered projects** — memoized projection based on active status
- **Available statuses** — auto-detected from data via `getAvailableStatuses()`
- **Crossfade** — opacity transition on container div: 150ms exit, URL push via setTimeout, then 200ms enter after re-render detects changed status
- **reduced-motion** — `window.matchMedia("(prefers-reduced-motion: reduce)")` check disables transition
- **Empty state** — "No {label} projects yet" with "Check back soon" message and conditional "Show all" Button (variant=outline) when filtered set is empty

### Task 2: Wire into page.tsx — Complete (3db9b20)

Added `import { ResearchGrid } from "@/components/research/ResearchGrid"` and a `<Section id="research">` block between Experience and Skills on the home page. Section title is "Research" with subtitle "Published & ongoing work in AI and ML" matching the UI-SPEC copy contract.
