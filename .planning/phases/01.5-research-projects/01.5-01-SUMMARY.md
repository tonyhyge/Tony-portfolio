### Task 2: Extend Badge component with statusColor CVA variant dimension

**Result: Complete**

- Added `statusColor` as a new variant key in `badgeVariants` CVA (separate from existing `variant`)
- 4 status color variants using oklch arbitrary values with light/dark mode pairs:
  - `in-progress`: amber tones (oklch 0.72 hue 72)
  - `submitted`: blue tones (oklch 0.55 hue 250)
  - `published`: green tones (oklch 0.58 hue 145)
  - `accepted`: purple tones (oklch 0.52 hue 290)
- No `defaultVariants` for statusColor — remains optional
- Badge component destructures `statusColor` between `variant` and `render`
- `badgeVariants({ variant, statusColor })` call updated to include the new dimension
- Existing `<Badge variant="secondary">Tech Tag</Badge>` in `experience-entry.tsx` unaffected

**Verification:** Runtime CVA output confirms all 4 status colors produce correct oklch arbitrary value classes. Combined `variant` + `statusColor` produces both class sets.
