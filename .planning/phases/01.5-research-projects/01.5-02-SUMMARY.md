# Phase 01.5 Plan 02: Research Components Summary

Created three leaf and composition components for the research section: PaperLinkChip (icon+label link), FilterTabs (URL-driven status filter with role=tablist ARIA), and ResearchCard (project card with badge, abstract expand, tags, and paper links).

## Tasks

### Task 1: PaperLinkChip — Complete (6aba465)

PaperLinkChip renders an anchor with lucide-react icon and label per link type (ExternalLink for arxiv/doi, FileText for pdf). Uses `target="_blank"` with `rel="noopener noreferrer"` for security (mitigates T-15-01). Named export only, no analog in codebase — built from primitives.

### Task 2: FilterTabs — Complete (4e8db55)

FilterTabs renders a `role="tablist"` wrapping `role="tab"` buttons with `aria-selected` for the active status. Always shows "All" first, then auto-detected statuses from data. Presentational component — receives `availableStatuses`, `activeStatus`, and `onFilterChange` as props. No hooks, no URL state — ResearchGrid (Plan 03) manages that. `formatStatusLabel` converts kebab-case statuses to display labels.

### Task 3: ResearchCard — Complete (12406f9)

ResearchCard is a composition component using Card, Badge, and PaperLinkChip. Renders: title + status Badge (with `statusColor` prop from Badge), venue/date line, optional thumbnail (D-15), abstract with `line-clamp-3` and conditional expand toggle when length > 150 chars (D-16), tags as Badge secondary chips, and paper links in CardFooter. Featured cards get `md:col-span-2` for bento grid treatment and `border-t-2 border-accent`.

## Verification

All tasks verified via `npm run build` — TypeScript compilation passes with zero errors.

## Deviations from Plan

None — all three components built exactly as specified in the plan.
