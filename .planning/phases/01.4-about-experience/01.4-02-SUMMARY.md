## Key Technical Details

### formatDate Utility (Task 1)
- Handles "Present" passthrough (returns `dateStr` as-is)
- Parses "YYYY-MM" input, creates Date with month-1 offset
- Formats via `toLocaleDateString("en-US", { year: "numeric", month: "long" })`
- Falls back to raw string if parsing fails
- Named export, preserves existing `cn` function

### ExperienceEntry Component (Task 2)
- **Timeline CSS pattern:** Container `relative pl-8 pb-8 last:pb-0`
- **Connector line:** `absolute left-[11px] top-3 bottom-0 w-px bg-border`, only rendered for non-last entries via `isLast` prop, `aria-hidden="true"`
- **Dot node:** `absolute left-0 top-[5px]` containing `size-2.5 rounded-full bg-accent ring-4 ring-background` -- ring masks the line behind the dot
- **Content panel:** `space-y-3` with role heading, company/location, formatted dates, description, highlights list, and tech tag Badge row
- **Tech tags:** `variant="secondary"` per D-10 (flat color), wrapped in `flex flex-wrap gap-1.5 pt-1`
- **No Card component** per D-06 (simple content panels, not Card header/footer pattern)

### ExperienceTimeline Component (Task 3)
- Imports `experiences` data array directly from `@/app/data/experience` per D-14
- Maps over entries, composes `ExperienceEntry` with `isLast` check
- Stable key: `${company}-${role}`
