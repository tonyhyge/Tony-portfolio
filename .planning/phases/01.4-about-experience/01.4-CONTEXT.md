<decisions>
## Implementation Decisions

### About Section Layout
- **D-01:** Split layout — narrative paragraph (left) + small stats block (right). Stats show: years of experience, lab affiliation, focus areas.
- **D-02:** Narrative tells the story: AI Student → AI Engineer → Researcher. Written as a 3-4 sentence personal narrative paragraph.
- **D-03:** Strictly professional — no personal interests or hobbies.
- **D-04:** No decorative imagery — text + stats only. The hero hexagon badge already provides visual interest above.

### Stats Block Content
- **D-05:** Three stats displayed: "3+ years experience", "AI Research Lab", "NLP & ML Systems". Rendered as a compact grid or row with small visual treatment (numbers prominent, labels subtle).

### Experience Timeline (Original)
- **D-06:** Hybrid connected timeline — vertical line with accent-colored dot nodes, but content blocks are styled as simple content panels (NOT full Card components with card header/footer pattern). Keeps the timeline feel without over-wrapping.
- **D-07:** Reverse chronological order — most recent role first (kept, applies per entry in groups).
- **D-09:** Tags appear as a Badge row below the description text (not inline in the header).
- **D-10:** Flat color — all tags use the same badge variant. No category-based color coding.

### Experience Data Structure (Updated for Concurrent Roles)
- **D-11 (updated):** Update `Experience` interface with:
  - Existing fields: `role`, `company`, `location`, `startDate`, `endDate`, `description`, `highlights`
  - Already present: `techTags: string[]`, `companyUrl?: string`
  - NEW: `track?: "industry" | "research"` — discriminates visual treatment for concurrent roles
  - NEW: `groupId?: string` — groups concurrent entries into a shared time-period row

### Concurrent Dual-Track Timeline (NEW)
- **D-15:** Entries sharing the same `groupId` render side-by-side in a dual-track layout. Entries without `groupId` render as single-column entries (backward compatible).
- **D-16:** Desktop uses 3-column CSS grid (`track | vertical-line | track`). Mobile (< 768px) collapses to `flex-col` stack with track badges (`Badge variant="outline"`).
- **D-17:** Grouping logic is a pure function (`groupExperiences()`) in the timeline component — single-pass O(n) scan, returns interspersed singles and groups.
- **D-18:** Zero new npm dependencies — all rendering uses existing project deps (Tailwind grid, shadcn Badge, lucide-react).
- **D-19:** `ExperienceEntry` receives optional `hideLine` prop. When inside a `DualTrackGroup`, the entry's left-side dot/line is suppressed on desktop (replaced by center column line and track-level dots).

### Dual-Track Visual Treatment
- **D-20:** Center vertical line renders between the two tracks on desktop. Each track's dot aligns to the inner edge facing the center line.
- **D-21:** Track badges ("Industry" / "Research") visible only on mobile. Desktop relies on side-by-side grid positioning for differentiation.
- **D-22:** Both tracks use the same accent-colored dot (per D-06). No per-track color differentiation in this phase.

### Section Integration
- **D-12:** About section renders in `#about` (already in `page.tsx`), uses Section component with "About" heading.
- **D-13:** Experience section renders in `#experience`, uses Section component with "Experience" heading.
- **D-14:** The existing `experience.ts` data file is consumed directly by the timeline component.

### Claude's Discretion
- Stats block exact layout (row vs grid, icon decoration)
- Timeline dot size, line thickness, and exact positioning
- Content block padding and hover effect (if any)
- Badge variant selection for tech tags — choose what matches the design system
- Exact spacing between timeline entries
- Responsive behavior — how the timeline collapses on mobile
- DualTrackGroup wrapper exact styling (center line thickness, gap between tracks)
- Grouping function location (inline in timeline file vs separate utility)

</decisions>

<specifics>
## Specific Ideas

- **Dual-track grid:** `grid-cols-[1fr_auto_1fr]` on desktop. Center column is `w-px bg-accent` vertical line.
- **Mobile collapse:** `flex flex-col gap-4` + track badges (`Badge variant="outline"`) above each entry.
- **Dots:** Single accent-colored dot per entry. In dual-track mode, dots align to inner edge facing center line.
- **Data:** AI Engineer (industry, groupId: "concurrent-1") + Research Assistant (research, groupId: "concurrent-1"). Both "Present".
- **Grouping:** Pure function `groupExperiences()` — returns `(Experience | Experience[])[]`, sorted reverse-chronologically.

</specifics>

<edge_coverage>
## Edge Coverage

### Covered
- Single entry without `groupId`: renders unchanged as single-column entry
- Dual entries with matching `groupId`: renders side-by-side in CSS grid
- Last item (group or single): bottom connector hidden
- Empty `techTags`: row omitted (existing behavior in ExperienceEntry)
- Missing `track` value: no badge label shown on mobile (treats as unlabeled)

### Backstop
- Group with only 1 entry (missing its pair): renders as single-column entry, logs warning
- Entries with `groupId` but no `track`: renders in dual-track layout without track badge
- `hideLine` prop when true: left-side dot and connector line are suppressed

### Unresolved
- Future 3+ concurrent entries (e.g., industry + research + teaching): CSS grid would need different template. Deferred until needed.

</edge_coverage>
