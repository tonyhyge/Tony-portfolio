    ```typescript
    import { cn } from "@/lib/utils";

    interface HexagonBadgeProps {
      /** Image source — when provided, renders an <img> clipped to hexagon instead of decorative SVG */
      src?: string;
      /** Alt text for the image (required when src is provided) */
      alt?: string;
      className?: string;
    }

    export function HexagonBadge({
      src,
      alt = "",
      className,
    }: HexagonBadgeProps) {
      const isDecorative = !src;

      return (
        <div
          className={cn(
            "relative shrink-0 size-[64px] md:size-[120px]",
            className
          )}
          aria-hidden={isDecorative || undefined}
        >
          {/* Outer hexagon clip container */}
          <div
            className="size-full"
            style={{
              clipPath:
                "polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)",
            }}
          >
            {src ? (
              <img
                src={src}
                alt={alt}
                className="size-full object-cover"
                style={{
                  clipPath:
                    "polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)",
                }}
              />
            ) : (
              /* Decorative SVG hexagon with gradient stroke */
              <svg
                viewBox="0 0 120 120"
                className="size-full"
                fill="none"
                xmlns="http://www.w3.org/2000/svg"
              >
                {/* Glow filter */}
                <defs>
                  <filter id="hex-glow" x="-50%" y="-50%" width="200%" height="200%">
                    <feGaussianBlur stdDeviation="4" result="blur" />
                    <feMerge>
                      <feMergeNode in="blur" />
                      <feMergeNode in="SourceGraphic" />
                    </feMerge>
                  </filter>
                  <linearGradient id="hex-stroke" x1="0%" y1="0%" x2="100%" y2="100%">
                    <stop offset="0%" stopColor="var(--accent)" />
                    <stop offset="100%" stopColor="var(--accent-secondary)" />
                  </linearGradient>
                </defs>
                {/* Subtle fill for visibility on dark bg */}
                <polygon
                  points="60,5 115,32.5 115,87.5 60,115 5,87.5 5,32.5"
                  fill="var(--color-card)"
                  fillOpacity="0.6"
                  stroke="url(#hex-stroke)"
                  strokeWidth="2.5"
                  filter="url(#hex-glow)"
                />
              </svg>
            )}
          </div>
        </div>
      );
    }
    ```

    Implementation details per UI-SPEC.md and CONTEXT.md:

    1. **D-06 — Decorative hexagon:** Uses SVG inline `polygon` with points defining a regular hexagon centered at (60,60) with radius 55: `60,5 115,32.5 115,87.5 60,115 5,87.5 5,32.5`. Per Claude's discretion (CONTEXT.md "Hexagon shape implementation: CSS clip-path or SVG inline"), SVG inline was chosen over CSS clip-path because it handles gradient border and glow filter more cleanly without extra wrapper elements.

    2. **Gradient border:** SVG `linearGradient` with `id="hex-stroke"` uses `--accent` (oklch(0.65 0.18 210) = cyan) to `--accent-secondary` (oklch(0.55 0.15 280) = violet). References CSS custom properties via `var(--accent)` and `var(--accent-secondary)` so colors are theme-aware per UI-SPEC.md Color section (Accent reserved for hexagon badge border/glow).

    3. **Glow effect:** SVG `<filter id="hex-glow">` with `feGaussianBlur stdDeviation="4"` creates a soft glow around the stroke. FeMerge composites the blur behind the source graphic. Per UI-SPEC.md "Hexagon badge border/glow effect" under accent reserved list.

    4. **D-07 — Right-side floating badge:** Positioning handled by consumer (Hero.tsx in Plan 04). This component renders at its default 64px mobile / 120px desktop.

    5. **D-08 — Swappable to photo:** When `src` prop is provided, renders `<img>` inside hexagon-clipped container. Both outer div and img use identical `clipPath: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)`. The `object-cover` class on img ensures photos fill the hexagon proportionally. When no `src`, renders decorative SVG. Swap without structural changes: `<HexagonBadge />` to `<HexagonBadge src="/photo.jpg" alt="Minh Pham" />`.

    6. **Sizing — 64px mobile, 120px desktop:** Uses `size-[64px] md:size-[120px]` — Tailwind responsive classes. Tailwind is mobile-first: `size-[64px]` is the default (mobile), `md:size-[120px]` overrides at md: and above (768px+). Per UI-SPEC.md "Hexagon badge 120px desktop / 64px mobile" and CONTEXT.md D-07 ("on mobile, shrinks to ~64px"). The SVG `viewBox="0 0 120 120"` makes it scale cleanly at any container size. Consumers can override by passing their own `className` with different sizing — cn() merges consumer className after defaults.

    7. **A11y (NFR2):** When decorative (no src), the outer div gets `aria-hidden="true"` per UI-SPEC.md Accessibility Contract "Hexagon badge has aria-hidden='true' (decorative)." When `src` is provided, `aria-hidden` is not set so screen readers discover the image with its `alt` text. `aria-hidden={isDecorative || undefined}` evaluates to `"true"` when decorative, `undefined` (attribute omitted) when not.

    8. **Background fill:** Semi-transparent `fill="var(--color-card)" fillOpacity="0.6"` provides subtle contrast on the dark gradient background, making the stroke glow more visible. `var(--color-card)` maps to card surfaces (dark: oklch(0.205 0 0) / light: oklch(1 0 0)), ensuring fill adapts to active theme.

    9. **Feature-based organization:** File lives in `src/components/hero/` per web/coding-style.md file organization rule (organize by feature, not by file type). SVG filter IDs are scoped within the SVG element, so multiple HexagonBadge instances on the same page won't conflict.
