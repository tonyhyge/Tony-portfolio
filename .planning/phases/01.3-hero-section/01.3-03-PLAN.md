    ```typescript
    "use client";

    import { useEffect, useState } from "react";
    import { ArrowDown } from "lucide-react";
    import { cn } from "@/lib/utils";

    export function ScrollIndicator() {
      const [opacity, setOpacity] = useState(1);

      useEffect(() => {
        let frameId: number;

        const handleScroll = () => {
          frameId = requestAnimationFrame(() => {
            const scrollY = window.scrollY;
            const fadeRange = 200;
            const newOpacity = Math.max(0, 1 - scrollY / fadeRange);
            setOpacity(newOpacity);
          });
        };

        window.addEventListener("scroll", handleScroll, { passive: true });
        handleScroll();

        return () => {
          window.removeEventListener("scroll", handleScroll);
          cancelAnimationFrame(frameId);
        };
      }, []);

      return (
        <div
          className="flex flex-col items-center gap-2"
          style={{ opacity }}
        >
          <span className="text-sm text-muted-foreground">
            Scroll to explore
          </span>
          <ArrowDown
            className={cn(
              "size-5 text-muted-foreground",
              "animate-bounce-subtle",
              "motion-reduce:animate-none",
            )}
          />
        </div>
      );
    }
    ```

    Implementation details per UI-SPEC.md and CONTEXT.md:

    1. **D-11 — Text + arrow:** "Scroll to explore" text rendered as `<span>` with `text-sm text-muted-foreground` (14px, muted gray per UI-SPEC.md Label typography role: "Scroll to explore secondary text"). ArrowDown icon from lucide-react at `size-5` (20px), same muted color.

    2. **D-12 — Fade on scroll within first 100-200px:** Uses `requestAnimationFrame` batched scroll listener with `passive: true` (per web/performance.md: avoid scroll handler churn). The fade range is 200px (`const fadeRange = 200`). Opacity decays linearly: at scrollY=0 → opacity=1, at scrollY=100 → opacity=0.5, at scrollY=200 → opacity=0. Below 200px, opacity clamps to 0 via `Math.max`. 200px was chosen per Claude's discretion (CONTEXT.md "Fade-on-scroll exact threshold" within D-12's specified 100-200px range; 200px gives the most gradual, natural feel). Uses `requestAnimationFrame` over raw scroll events per RESEARCH.md anti-pattern: avoids layout thrashing by batching style writes to the next paint cycle.

    3. **Controlled via `style={{ opacity }}` not className:** Using inline style for opacity allows smooth per-frame updates without class toggling. The fade is tied to scroll position, not a binary visible/hidden state. CSS transition is NOT used here because the opacity changes continuously with scroll — a transition would create a lagging effect. The `useEffect` with scroll listener is the correct approach for scroll-driven values.

    4. **Initial state:** `const [opacity, setOpacity] = useState(1)` — fully opaque on first render. The effect calls `handleScroll()` immediately to ensure the opacity reflects the current scroll position at mount (handles the edge case where user loads page mid-scroll).

    5. **Pulse animation:** Arrow uses `animate-bounce-subtle` (2s translateY 0→4px cycle). Per UI-SPEC.md Interaction Contracts: "gentle pulse animation on arrow (CSS `transform: translateY(4px)` cycle, 2s)". `motion-reduce:animate-none` disables the pulse for users who prefer reduced motion per UI-SPEC.md Accessibility Contract and web/performance.md animation performance: "Animate compositor-friendly properties only" — `translateY` is compositor-friendly.

    6. **D-12 — No JS timer:** The scroll fade uses a scroll event listener (not setTimeout/setInterval). No JS timer is involved. The pulse animation is CSS-only (via `animate-bounce-subtle`). Per CONTEXT.md D-12: "CSS opacity transition, no JS timer."

    7. **A11y (NFR2):** Per UI-SPEC.md Accessibility Contract: "Scroll indicator pulse disabled. Fade-on-scroll retained (functional — not decorative)." The fade-on-scroll is functional (it reduces visual clutter after the user has started browsing), so it stays in reduced-motion mode. Only the decorative pulse animation is suppressed.

    8. **"use client":** Required because `useEffect` and `useState` are used for the scroll-driven opacity state.
